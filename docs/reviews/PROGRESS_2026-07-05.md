## 2. Maintainer decisions recorded (`MAINTAINER_DECISIONS.md`)

1. Hybrid reader is essential; api/log stay standalone; **hybrid is now the default**
   for routed reads. Rationale: api/log staleness is complementary; for changes the
   target is known, so one admissible witness suffices.
2. Backlash = simple go-and-return **procedure outside acquisition** in the driver;
   exposed as a Leica acquisition option in the controller. (Not yet implemented.)
3. Test strategy: three tiers (hardware / mock hardware / offline); adapter must be
   tested **through the controller**.
4. Orphan test scripts may be deleted (done).
5. `experimental/` may be reworked without bench validation.
6. `confirmed` is best-effort — 3 retries then report unconfirmed and continue —
   **except acquire's idle gate, which must be confirmed** (and still needs a real
   deadline so a dead LAS X errors instead of hanging).
7. **Limits enforced at the lowest layer** (command wrappers, so nothing on top can
   bypass); **no bundled default limits for enforcement** (templates only; the
   `set_stage_limits` notebook is the file factory; connect verifies placement);
   hardcoded physical backstop for the motoric stage; explicit `null` = deliberately
   unlimited, absent key fails closed; calibration co-located but applied at the
   objective-change path. Implemented (`474c3a6`, `b90b92f`); plan +
   two review docs under `docs/design/`.
